# Contributing to CubeSolver

Thank you for your interest in contributing! 🎉
This project is designed to be educational and beginner-friendly, so contributions should prioritize **clarity and readability**.

---

## 🧭 Project Goals

* Teach students how to program a Rubik’s Cube solver
* Provide clean, understandable implementations of algorithms
* Support robotics and computer vision integration

---

## 🚀 Getting Started

1. Fork the repository
2. Clone your fork:

   ```bash
   git clone https://github.com/YOUR_USERNAME/cubesolverCampFull.git
   ```
3. Navigate into the project:

   ```bash
   cd cubesolverCampFull
   ```
4. Run the project:

   ```bash
   ./gradlew run --args="O Y N"
   ```

---

## 🛠 Making Changes

### Create a branch

```bash
git checkout -b feature/your-feature-name
```

---

### Coding Guidelines

* Write clear, readable code
* Prefer simple logic over clever tricks
* Add comments for non-obvious steps
* Follow existing naming conventions

Example:

```java
// Move piece to buffer
// Apply swap algorithm
// Undo setup moves
```

---

### Documentation

If you:

* Add a feature → document it
* Add a method → include comments
* Change behavior → update README/docs

---

## 🧪 Testing

Before submitting:

* Ensure the project builds
* Run the program with different arguments
* Verify solving still works

---

## 📥 Pull Requests

1. Push your branch:

   ```bash
   git push origin feature/your-feature-name
   ```
2. Open a Pull Request
3. Clearly describe:

   * What you changed
   * Why you changed it
   * Any tradeoffs

---

## 🐛 Reporting Issues

When creating an issue, include:

* What you expected to happen
* What actually happened
* Steps to reproduce
* Screenshots/logs if applicable

---

## 💡 Good First Contributions

* Improve documentation
* Add comments to complex code
* Create example programs
* Refactor confusing methods
* Add beginner-friendly explanations

---

## ⚠️ Guidelines for This Project

Because this is an educational project:

* Avoid over-engineering
* Avoid unnecessary abstractions
* Keep logic easy to follow
* Prefer explicit steps over compact code

---

## 🙌 Community

Be respectful and constructive.
This project is meant to help people learn.

---

Thanks for contributing! 🚀
