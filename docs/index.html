Introduction

CubeSolver is an integrated robotics system designed to automatically solve Rubik's Cubes using computer vision and precise motor control. The software runs on a Raspberry Pi and interfaces with a camera and stepper motors to observe the cube, compute a solving sequence, and physically execute moves.

The system is designed to be modular and extensible. Each component — vision, solver logic, and motor control — is implemented as a separate class so that new solving methods, hardware configurations, or camera systems can be added without rewriting the entire system.

System Pipeline

Camera → Vision → Cube State → Solver → Motor Controller → Physical Movement

Each stage in the pipeline transforms the cube state into a new representation. The camera captures images, the vision system interprets colors, the solver computes moves, and the motor controller executes those moves.

Installation

This section explains how to install and configure the CubeSolver system on a Raspberry Pi. These steps assume basic familiarity with Linux command line usage and Git.

Clone Repository

git clone https://github.com/randomcuber/cubesolverCampFull
cd cubesolverCampFull

Install Dependencies

pip install opencv-python
pip install numpy

OpenCV is required for computer vision operations such as color detection and image capture. NumPy is used for numerical processing and matrix operations within the solver.

Run the Program

python main.py

Reference

The Reference section documents the internal architecture of the CubeSolver system. Each major class is explained in detail, including responsibilities, methods, and how the components interact with each other.

MotorController Class

The MotorController class is responsible for controlling the motors that rotate cube faces. It converts logical moves (such as R or U') into electrical signals that drive stepper motors. 

controller = MotorController()
controller.rotate_face("R")

Internally, the controller calculates step counts, direction, and timing to ensure accurate and safe movement of the cube.

OpenCVRaspberryPiCamera Class

This class captures images from the Raspberry Pi camera and detects cube colors using OpenCV. It converts raw camera frames into structured cube state data that can be passed to the solver. 

camera = OpenCVRaspberryPiCamera()
state = camera.scan_cube()

Usage

This section explains how to operate the system, configure solver behavior, and run commands from the terminal.

Navigate to Project

cd cubesolverCampFull

Run Program

python main.py

Select Solving Method

solver = Solver(method="CFOP")

Different solving methods can be implemented and selected dynamically using the method parameter. This allows experimentation with different solving algorithms.

Tuning

Tuning is required to ensure accurate cube detection and reliable motor operation. Environmental factors such as lighting, camera angle, and cube position can affect detection accuracy.

Coordinate Tuning

self.coordinates = [
    (120, 80),
    (200, 80),
    (280, 80)
]

Each coordinate represents the center of a cube sticker. Adjusting these values allows the camera to correctly sample colors from the cube.

Calibration Command

python main.py --calibrate

Contributing

Developers can extend the CubeSolver system by adding new solving algorithms or supporting new hardware configurations.

Add New Solver

class MySolver:
    def solve(self, cube):
        return ["R", "U", "R'"]

New solver implementations should follow the same interface as existing solver classes so that they can be integrated without modifying other components.
